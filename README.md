# Chillist Backend

REST API for trip/event planning with shared checklists.

## README-first workflow for Cursor

1. Read this `README.md` first.
2. Identify the exact feature area from the file map and "Where to find X" sections below.
3. Open only those files (do not scan the whole repo).
4. Use external docs/spec links only when this README points to a missing detail.

## System overview

- Runtime: Node.js 20+, Fastify 5, TypeScript (ESM)
- Validation and OpenAPI: Zod schemas registered in Fastify and referenced with `$ref`
- Persistence: PostgreSQL via Drizzle ORM
- Auth model: JWT verification via Supabase JWKS (`Bearer` auth) plus invite-token guest flows
- API docs: Swagger UI at `/docs`, generated spec at `docs/openapi.json`

Request flow (high-level):
1. `src/index.ts` creates DB connection and starts Fastify.
2. `src/app.ts` registers schemas, plugins (auth/guest/websocket), hooks, then routes.
3. Route handlers in `src/routes/*.ts` validate against schemas and use `fastify.db`.
4. OpenAPI is generated from registered route schemas (`npm run openapi:generate`).

## Quick start

```bash
npm install
cp .env.example .env
npm run db:migrate
npm run dev
```

Server runs at `http://localhost:3333`. Swagger UI at `http://localhost:3333/docs`.

## Standard workflow for adding or editing endpoints

1. Identify domain route file in `src/routes/`.
2. Add or update schema in `src/schemas/` (do not inline complex schemas in route files).
3. Register schema in `src/schemas/index.ts` if it is new.
4. Implement or update the route handler in `src/routes/<domain>.route.ts`. If the handler needs reusable business logic (e.g., creating a participant, syncing profiles), put it in `src/services/`. Services are pure functions that receive `db` as an argument (DI) — no Fastify coupling.

5. If needed, update plugin/type wiring (`src/plugins/*`, `src/types/fastify.d.ts`, `src/app.ts` route/plugin registration).
6. Add or update tests in `tests/integration/` (and unit tests where relevant).
7. Regenerate OpenAPI: `npm run openapi:generate`.
8. Run validation: `npm run typecheck && npm run lint && npm run test:run`.

## Common scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start Fastify in watch mode |
| `npm run dev:local` | Load `.env.local`, start Docker DB, migrate, seed, run dev server |
| `npm run db:migrate` | Apply pending migrations |
| `npm run db:seed` | Seed local database |
| `npm run test:run` | Typecheck + lint + test (single run) |
| `npm run openapi:generate` | Generate `docs/openapi.json` |
| `npm run openapi:validate` | Validate generated OpenAPI |
